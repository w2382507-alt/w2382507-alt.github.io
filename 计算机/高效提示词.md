（一）基础通用技巧
**Zero-shot Prompting（零样本提示）**
核心：无示例、无辅助信息，直接给 LLM 下达指令；

**K-shot Prompting（K 样本提示）**
别称：上下文学习（In-context Learning）；
核心：下达指令时搭配 1/3/5 个实证最优示例，引导 LLM 对齐输出标准；

（二）推理增强技巧
**Chain-of-Thought Prompting（思维链提示，CoT）**
核心：引导 LLM 显式输出推理步骤，解决多逻辑步骤的思维断层问题；
子形式：Multi-shot CoT（带推理步骤的示例引导）、Zero-shot CoT（仅用 “Let's think step-by-step” 指令引导）；

**Self-consistency Prompting（自一致性提示）**
核心：搭配 CoT 使用，对同一任务多次采样输出，取出现频率最高的结果；

（三）工程化进阶技巧（专业实操，适配企业 / 复杂工程场景）
**Tool Use（工具使用）**
核心：允许 LLM 根据任务需求调用外部工具 / 系统（如测试脚本、第三方接口、检索工具）；

**Retrieval Augmented Generation（RAG，检索增强生成）**
核心：为 LLM 注入专属上下文数据 / 权威资料，无需重训即可更新知识；

**Reflexion（反思优化）**
核心：引导 LLM 对自身输出复盘、批判，结合环境反馈（如报错、测试不通过）多轮迭代修正；

（四）
- **Clear prompting**
	Give prompt to someone with minimal context and if they're confused an LLM will be too
- Use **role prompting** aggressively to make system prompts more powerful
